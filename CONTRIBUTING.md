# Contributing to CarbonLite

We welcome contributions! As part of the Prompt Wars Virtual Hackathon, this repository serves as a model for clean, maintainable code.

## Code Quality Standards
- **ESLint**: We enforce strict linting rules. 
- **Prettier**: All code must be formatted with Prettier (
pm run format).
- **JSDoc**: Every function must be documented using JSDoc.
- **Testing**: All mathematical calculations must have corresponding unit tests in 	ests/. Run tests via 
pm test.

## Accessibility (WCAG 2.1 AA)
Any UI changes must maintain strict WCAG compliance:
- Semantic HTML (<main>, <nav>, <button>)
- Explicit ria-labels on iconic buttons.
- Proper contrast ratios.
